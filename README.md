# GRC Security Gap Assessment

## Overview

This project presents a cybersecurity gap assessment conducted for a simulated third-party logistics (3PL) organization using the NIST Cybersecurity Framework (CSF) 2.0.

The purpose of this assessment was to evaluate the organization's current cybersecurity posture, identify security gaps, assess organizational risk, and develop prioritized recommendations to improve overall cybersecurity maturity.

This project demonstrates practical Governance, Risk, and Compliance (GRC) skills through the creation of a professional security assessment report, formal risk register, and remediation roadmap.

---

# Objectives

* Evaluate cybersecurity controls using the NIST Cybersecurity Framework (CSF) 2.0
* Identify organizational security gaps
* Perform a structured cybersecurity risk assessment
* Prioritize risks based on business impact and likelihood
* Develop actionable remediation recommendations
* Improve organizational cybersecurity maturity

# Key Findings

The assessment identified several opportunities to improve the organization's cybersecurity maturity.

Highest-priority findings included:

* Multi-Factor Authentication (MFA) not fully implemented
* Limited centralized security monitoring
* Incomplete asset inventory
* No formal Incident Response Plan
* Lack of regular vulnerability assessments
* Limited third-party risk management process

---

# Recommendations

Priority recommendations include:

* Deploy organization-wide Multi-Factor Authentication
* Implement centralized log monitoring using a SIEM platform
* Establish a formal Incident Response Plan
* Maintain a centralized asset inventory
* Perform routine vulnerability assessments
* Improve employee security awareness training
